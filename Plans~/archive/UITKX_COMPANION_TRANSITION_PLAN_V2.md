# UITKX Companion File Transition — Full Implementation Plan V2
# `hook` and `module` Keywords with Option C HMR

> **Vision:** Replace companion `.cs` files with `.uitkx` files using `hook`
> and `module` keywords. All reactive and utility code lives inside the UITKX
> ecosystem — IDE support, HMR, formatting, diagnostics, and coloring come free.
>
> **HMR Strategy:** Option C — delegate indirection with `#if UNITY_EDITOR`
> guards. Zero cascade recompilation. Explicit return types via `->` syntax.

## 1. Syntax Reference

### 1.1 Hook Declaration

```uitkx
@namespace MyApp.Hooks

hook useCounter(int initial = 0) -> (int, Action) {
    var (count, setCount) = useState(initial);
    var increment = useCallback(() => setCount(count + 1), count);
    return (count, increment);
}
```

**Rules:**
- `-> ReturnType` is **required** when the hook returns a value
- No `->` means void hook (uses `Action<>` for HMR indirection)
- Body is pure C# — no markup/JSX
- Hook aliases work (`useState` → `Hooks.UseState`)
- Multiple hooks per file allowed

### 1.2 Module Declaration

```uitkx
@namespace MyApp.UI

module Home {
    public static Style HeaderStyle => new Style { (FlexDirection, "row") };
    public static void LogAction(string msg) => Debug.Log(msg);
}
```

**Rules:**
- `module ComponentName` → generates `partial class ComponentName` (extends component)
- `module OtherName` → generates standalone `public partial class OtherName`
- Body is pure C# — no markup/JSX, no hooks
- No hook aliases, no `[HookSignature]`
- Multiple modules per file allowed

### 1.3 Preamble Directives

| Directive | `component` | `hook` | `module` |
|-----------|:-----------:|:------:|:--------:|
| `@namespace` | Yes | Yes | Yes |
| `@using` | Yes | Yes | Yes |
| `@uss` | Yes | **No** | **No** |
| `@props` | Yes | **No** | **No** |
| `@key` | Yes | **No** | **No** |
| `@inject` | Yes | **No** | **No** |

### 1.4 File Naming Conventions

| File | Contains | Use Case |
|------|----------|----------|
| `Counter.uitkx` | `component Counter { ... }` | UI component |
| `useCounter.uitkx` | `hook useCounter(...) -> ... { ... }` | Standalone hook |
| `Counter.hooks.uitkx` | `hook useX(...) -> ... { ... }` | Component-associated hooks |
| `Counter.styles.uitkx` | `module Counter { ... }` | Styles extending component |
| `SharedUtils.uitkx` | `module SharedUtils { ... }` | Reusable utilities |

All end in `.uitkx` — existing `*.uitkx` file discovery works unchanged.

### 2.3 Generic Hook — MethodInfo + Cached Delegate

> **Problem:** Can't declare `static Func<string, T, (T, Action<T>)>` because
> `T` doesn't exist at class scope.
>
> **Solution:** Store a `MethodInfo` reference to the open generic body method.
> The trampoline — which *is* generic and knows `T` — closes it at call time
> via `MakeGenericMethod` and caches the resulting typed delegate per `T`.
> Zero overhead on subsequent calls.

**Input:**
```uitkx
hook useLocalStorage<T>(string key, T defaultValue) -> (T, Action<T>) {
    var (value, setValue) = useState(defaultValue);
    return (value, setValue);
}
```

**Generated:**
```csharp
#if UNITY_EDITOR
    internal static System.Reflection.MethodInfo __hmr_useLocalStorage = null;
    internal static readonly ConcurrentDictionary<Type, Delegate> __hmr_useLocalStorage_cache = new();
#endif

[HookSignature("UseState")]
public static (T, Action<T>) useLocalStorage<T>(string key, T defaultValue)
{
#if UNITY_EDITOR
    if (HmrState.IsActive && __hmr_useLocalStorage != null)
    {
        var del = (Func<string, T, (T, Action<T>)>)__hmr_useLocalStorage_cache
            .GetOrAdd(typeof(T), t =>
            {
                var closed = __hmr_useLocalStorage.MakeGenericMethod(t);
                return Delegate.CreateDelegate(
                    typeof(Func<string, T, (T, Action<T>)>), closed);
            });
        return del(key, defaultValue);
    }
#endif
    return __useLocalStorage_body(key, defaultValue);
}

private static (T, Action<T>) __useLocalStorage_body<T>(string key, T defaultValue)
{
#line 3 "useLocalStorage.uitkx"
    var (value, setValue) = Hooks.UseState(defaultValue);
    return (value, setValue);
#line hidden
}
```

**HMR swap for generic hooks:**
```csharp
// In HookDelegateSwapper:
var newBody = newType.GetMethod("__useLocalStorage_body", NonPublic | Static);
originalType.GetField("__hmr_useLocalStorage").SetValue(null, newBody);
// Clear cache so closed delegates get recreated with new body
((ConcurrentDictionary<Type, Delegate>)originalType
    .GetField("__hmr_useLocalStorage_cache").GetValue(null)).Clear();
```

**Performance:** First call per `T` pays ~1-2µs for `MakeGenericMethod` +
`CreateDelegate`. Subsequent calls hit the cached typed delegate — same speed
as non-generic hooks. All stripped in builds via `#if UNITY_EDITOR`.

### 3.3 Step-by-Step Parser Changes

**Step 1 — `s_topLevelKeywords` (line 33):** add `"hook"`, `"module"`.

**Step 2 — `LooksLikeFunctionStyleComponent()` (line 687):** change final return:
```csharp
return TryReadKeywordAt(source, i, "component")
    || TryReadKeywordAt(source, i, "hook")
    || TryReadKeywordAt(source, i, "module");
```

**Step 3 — `TryParseFunctionStyle()` (line 131):** multi-keyword detection.
Replace single `component` check (line 172) with:
```csharp
string? keyword = null;
if (TryReadKeyword(source, ref i, "component")) keyword = "component";
else if (TryReadKeyword(source, ref i, "hook")) keyword = "hook";
else if (TryReadKeyword(source, ref i, "module")) keyword = "module";
else return false;
```

Then branch: `component` → existing flow; `hook` → Step 4; `module` → Step 5.

**Step 4 — Hook parsing branch:**
```csharp
SkipSpaces(source, ref i);
TryReadIdentifier(source, ref i, out string hookName);    // camelCase OK
string? genericParams = null;
if (i < source.Length && source[i] == '<')
    genericParams = ReadGenericParams(source, ref i);       // new helper
SkipSpaces(source, ref i);
var hookParams = ParseFunctionParamList(...);               // reuse existing
// -> ReturnType: read everything between -> and {
string? returnType = TryReadArrowReturnType(source, ref i, ref line);
// { body }
TryReadBalancedBlock(source, bodyOpen, out int bodyClose);  // reuse existing
// Extract body, add to hooks list
```

**Critical: `TryReadArrowReturnType` does NOT use `TryReadTypeName`.** It reads
from `->` to `{` tracking balanced parens/angles:
```csharp
if (source[i] == '-' && source[i+1] == '>') {
    i += 2; SkipSpaces;
    int start = i;
    int parenDepth = 0, angleDepth = 0;
    while (i < source.Length) {
        if (source[i] == '(') parenDepth++;
        else if (source[i] == ')') parenDepth--;
        else if (source[i] == '<') angleDepth++;
        else if (source[i] == '>') angleDepth--;
        else if (source[i] == '{' && parenDepth == 0 && angleDepth == 0) break;
        i++;
    }
    returnType = source[start..i].Trim();
}
```

**Step 5 — Module parsing branch:**
```csharp
SkipSpaces; TryReadIdentifier → moduleName;
SkipWhitespaceAndNewlines; TryReadBalancedBlock → body;
modules.Add(new ModuleDeclaration(...));
```

**Step 6 — Multi-declaration loop.** Wrap Steps 3-5 in a `while` loop. After each
declaration body closes, advance `i` past `}`, skip trivia, try next keyword.

**Step 7 — Preamble validation.** After keyword detected, if `hook`/`module`
and `ussFiles.Count > 0` → emit UITKX2210.

## 8. HMR Implementation (Option C)

### 8.1 Architecture

```
Hook save → FileWatcher → Controller.ProcessFileChange()
  → DirectiveParser (via reflection) detects hook declarations
  → Compiler.CompileHooks(): HmrHookEmitter → CompileSources() → Assembly
  → HookDelegateSwapper.SwapAll():
      1. Find original container type in AppDomain
      2. originalType.GetField("__hmr_hookName", NonPublic|Static)
      3. newType.GetMethod("__hookName_body", NonPublic|Static)
      4. Delegate.CreateDelegate(fieldType, newBody)
      5. originalField.SetValue(null, newDelegate)
  → TriggerGlobalReRender() (all renderers)
```

### 8.2 Key Difference from Component HMR

Component HMR (UitkxHmrDelegateSwapper.SwapAll, line 19):
- Walks ALL fiber trees → finds matching `FiberNode` → swaps `fiber.TypedRender`
- Delegate: `Func<IProps, IReadOnlyList<VirtualNode>, VirtualNode>`

Hook HMR:
- Swaps ONE **static field** per hook via reflection — no fiber walking
- After swap, global re-render triggers all components to re-execute with new logic

### 8.3 Controller (UitkxHmrController.cs)

In `ProcessFileChange()` after DirectiveParser result:
- If `ComponentName == null` and hooks exist → `CompileHooks()` → `HookDelegateSwapper.SwapAll()` → `TriggerGlobalReRender()`
- `TriggerGlobalReRender()` iterates `EditorRootRendererUtility.GetAllRenderers()` and `RootRenderer.AllInstances`, invokes `ComponentState.OnStateUpdated` on root fibers

### 8.4 HookDelegateSwapper.cs (New)

Iterates hook declarations via reflection (`GetProp`). For each hook:

**Non-generic hooks:** finds `__hmr_hookName` `Func<>` field on original type,
creates new delegate from HMR assembly's body method, swaps via `FieldInfo.SetValue`.

**Generic hooks:** finds `__hmr_hookName` `MethodInfo` field on original type,
sets it to the new body `MethodInfo` from HMR assembly, then clears the
`__hmr_hookName_cache` `ConcurrentDictionary` so closed delegates get lazily
rebuilt with the new body on next call.

### 8.5 Compiler (UitkxHmrCompiler.cs)

New `CompileHooks()` method — much simpler than `Compile()`:
- No AST parsing, no lowering, no markup
- Just `HmrHookEmitter.Emit()` → `CompileSources()` (reuses existing Roslyn path)

### 8.6 HmrHookEmitter.cs (New)

Generates only the body method (delegate field + trampoline already exist in
original assembly). Uses reflection (`GP<T>()`) to access DirectiveSet
properties, same pattern as existing `HmrCSharpEmitter`.

### 8.7 HMR Compiler Reflection

HMR compiler accesses `DirectiveParser.Parse()` return via reflection. New
`HookDeclarations` is `ImmutableArray<HookDeclaration>` which implements
`IEnumerable` — reflection iteration works. Each `HookDeclaration` record's
public properties (`Name`, `Body`, `ReturnType`, etc.) accessible via `GetProp()`.

### 8.8 Module HMR

No delegate indirection. Module changes trigger domain reload (acceptable —
modules contain constants/styles/utilities, not reactive logic).

## 12. Risk Analysis

### 12.1 Generic Hooks — MethodInfo Indirection

Generic hooks use `MethodInfo` + `ConcurrentDictionary<Type, Delegate>` cached
delegate instead of a direct `Func<>` field. First call per `T` after HMR swap
pays ~1-2µs for `MakeGenericMethod`; subsequent calls are direct typed delegate
invocation. All stripped in builds.

### 12.2 `hook`/`module` as Identifiers

Only keywords at top-level. `TryReadKeyword` word boundary checks (line 1343)
prevent false matches inside C# code.

### 12.3 `TryReadTypeName` vs Tuple Return Types

`TryReadTypeName` (line 578) fails on `(int, Action)`. Solution: `->` parser reads
everything between `->` and `{` with balanced paren/angle tracking. Handles all
C# types including tuples, generics, arrays, nullable.

### 12.4 HMR Reflection Access

`ImmutableArray<HookDeclaration>` is `IEnumerable` — reflection can iterate.
Record properties are public. Same pattern as existing `FunctionParams` access.

### 12.5 Container Class Collisions

`partial class` merging handles this. `Counter.hooks.uitkx` + `Counter.utils.uitkx`
both produce `CounterHooks` partial — intentional merging.

### 12.6 Thread Safety

HMR (via `EditorApplication.delayCall`) and UI Toolkit rendering both run on
Unity main thread. No race condition.
